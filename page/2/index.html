Java 虚拟机（JVM）配置选项说明（JDK8+）

发表于 2022-08-12 更新于 2022-10-15 分类于 article

标准选项（Standard Options）

这些是Java虚拟机（JVM）的所有实现都支持的最常用的选项。

选项和默认值 描述 示例
-agentlib:libname[=options] 加载本地的代理类 -agentlib:jdwp=transport=dt_socket,server=y,address=8000
-agentpath:pathname[=options] 加载绝对路径的本地的代理类
和-agentlib类似，区别是使用绝对路径
-client 使用Java HotSpot客户端虚拟机
-Dproperty=value 设置系统属性的值
注意：属性名不可以有空格，属性值可以有空格（有空格的时候用英文双引号 " 括起来，如："foo bar"）
-disableassertions[:[packagename]…|:classname]
-da[:[packagename]…|:classname]		 禁用断言
默认情况下所有包中所有类里面的断言都是被禁用的		 -da
-da:com.wombat.fruitbat.Brickbat MyClass
-disablesystemassertions
-dsa		 禁用所有系统类中的断言
-enableassertions[:[packagename]…|:classname]
-ea[:[packagename]…|:classname]		 启用断言
默认情况下所有包中所有类里面的断言都是被禁用的
-enablesystemassertions
-esa		 启用所有系统类中的断言
-help
-?		 在不运行JVM的情况下，显示java命令的使用信息
-jar filename 执行封装成JAR文件的程序 java -jar foo.jar
-javaagent:jarpath[=options] 加载指定的Java语言代理
-jre-restrict-search 在版本搜索中包括用户私有的JRE
-no-jre-restrict-search 在版本搜索中排除用户私有的JRE
-server 使用Java HotSpot服务器虚拟机
-showversion 显示版本信息，并且继续执行程序
-splash:imgname 通过指定的图片来显示启动画面 -splash:images/splash.gif
-verbose:class 显示每个加载的类的信息
-verbose:gc 显示每次垃圾回收（GC）事件的信息
-verbose:jni 显示本地方法（native methods）的使用信息和其它本地接口（JNI, Java Native Interface）的活动信息
-version 显示Java版本信息，然后退出程序
-version:release 指定运行程序使用的发行版本 -version:”1.6.0_13 1.6* & 1.6.0_10+”

黑夜的引路人

发表于 2022-05-11 更新于 2022-10-15 分类于 article

这件事发生在我小时候，我那时还没上学，跟着我爸在河南驻马店做炸油条的生意。

夏季多雨，最近更是下了很久的雨。这几天刚放晴，家里的生意有点忙，大人也无暇照看我，我爸就想把我送到临近的姑姑家，让我在姑姑家待一段日子。

这天晚上，我们收摊回家，吃完晚饭，天已经黑了。我爸带着我，出发去姑姑家。

那时候我还小，我只记得从我们做生意的地方，到我姑姑家，需要过一条河，还有几块田野。田野是一块一块的，是因为田地很多，而且由很多横竖的小路分隔开，庄稼还没有种上，感觉小路略微高出田地。

过河的话，是有个老人家专门撑船载客的，过完河之后，就是几块大的田野了。

过完河之后，在大路上走了一会儿，走到前面的路口，需要穿过田野了，这时候我们都愣住了。

前一阵儿一直在下雨，没想到雨水把田地都淹了，一眼望去都是水，根本分不清田间的小路在哪，只能看到大路边上的小路路口。

田间的小路都已经被水覆盖了，看不到底。邻村的大路还很远，天色已晚绕过去还要很久，我爸觉得应该不是很深，想试试趟过去。

我爸背着我，开始往前走。刚开始水只到小腿那里，但是越往里面走水越深，往里面走个几十米，水就到大腿根了。

天已经很黑了，月光也不怎么亮。我爸站在水里面，一时也不知道该怎么好，鞋子下水都已经湿了，水路都走了小半，往回看都是水汪汪的一片，退回去已经分不清来时的路，绕到大路也不知道还要多久，但是往前走的话，不知道前面会不会更深，万一谁家田里有口井，就麻烦了。

另一方面，我爸背着我，也害怕我睡着了，一直在跟我说话：“儿子，你别睡着了，帮我看着点路”。我说：“爸爸，你看前面有个人影，也在趟水往前走呢”。我爸说他也看见了，估计是个本村的熟路人。路人穿白衣服，夜色当中依稀可辨，我爸就赶紧跟上去，在他后面十来米的地方，紧紧地跟着，看看他从哪趟的水，我爸也从哪趟。

就这样，终于来到了距离大路不远的地方，这时候我的鞋子掉了，我跟我爸说让他帮我找一下，我爸就弯下腰找鞋子，鞋子也没找到，我们抬起头，突然发现，那个穿白衣服的人不见了，我爸背着我赶紧上了大路。

我们以为那个人进了路边的林子，我爸就背着我走近林子，想要过去打个招呼，说声谢谢。刚走进林子，没看到那个人，看到林子里有好几个坟，像是个老坟园，吓得我们赶紧跑回大路，沿着大路跑回姑姑家了。

到了姑姑家，说起此事，大家都觉得挺怪的，我爸也是等第二天天亮了，才回去做生意的地方。

后来，跟当地人聊天说起这事儿，当地人也说，那附近之前就发生过一些奇怪的事情，我们听了都有点后怕。

但是很感激那晚的引路人，多亏了他的帮忙，我们才走出了那片田野。

写在最前面的话

发表于 2022-04-10 更新于 2022-10-15 分类于 article

迪士尼的动画永远有着很大的魅力。

最近补了好多动画，很多都是很早之前的动画《美女与野兽》、《阿拉丁神灯》等等。总是让人欲罢不能。

最近补了一部影片《疯狂动物城》，讲述了一只叫朱迪的兔子有着当警察的梦想，身边所有的人包括父母都觉得这个理想太不现实了，一只弱小的兔子怎么能当警察呢?

可是朱迪并不气馁，通过自己的努力成为她那一届最出色的警察学院的毕业生，并且被派向警局工作。

当朱迪兴高采烈的前往警察厅工作却发现一切并非想象那样，自己并不受重视，只是安排去管交通做一些贴罚单等简单的工作，甚至在上班第一天都被一只“狡猾”的狐狸欺骗。

可是朱迪没有被打倒，努力工作抓住机会，途中竟然和这只“骗人”的狐狸成为信任的搭档，一起破解了动物城最棘手的案件。

看完之后我感触很深，疯狂动物城果然疯狂，一只兔子能靠自己的努力成为一名警察，狡猾的狐狸也值得相信，成为最佳拍档。

树懒都能够颠覆最开始的认知飙车行驶，还有什么会超出想象呢？

这个公众号是我和我男朋友共同创建的，没有规划，不知道要写什么，也不知道未来会发生什么事。

或许我们很勤奋每天更新内容，当然可能没有什么营养只是流水文，或者产出的东西对你没有什么帮助，也或许我们不定时更新，或者很长时间都很荒芜，也可能我们的文字很稚嫩，分享的东西没有很新潮，亦或许存在错误。

但是谁知道兔子下一刻会怎么样呢？或许它就是一直吃胡萝卜的兔子，但说不定它慢慢会成为兔子警察呢？

这个公众号会见证我们在一起的过程，也希望会见证接下来的里程。

感谢阅读我们文章关注我们的每一个朋友，希望我们一起成长、见证彼此的进步！

程序源码

# 使用示例：progress 1 100
# 参数1：当前数值
# 参数2：总数值
progress () {
    pro=`awk 'BEGIN{printf "%d",('$1'/'$2')*100}'`
    spaces=`seq -s '#' $((${pro} +1)) | sed 's/[0-9]//g'`
    if [ "${pro}" != "100" ]; then
        printf "[%-100s] %d%% \r" "${spaces}" "${pro}"
    else
        printf "[%-100s] %d%% \n" "${spaces}" "${pro}"
    fi
}

Java中的监视器（monitor）是什么？

发表于 2021-11-30 更新于 2022-10-15 分类于 article

前言

如果查看Java源码java.lang.Object，就能够看到好多地方提到监视器（monitor），都是出现在描述线程竞争关系的时候，比如Object.notify方法和Object.wait方法。

简要描述

监视器是一个用来保证多个线程安全访问共享数据的机制。

监视器和锁的区别？

严格地讲，“监视器”和“锁”这两个术语表达的含义是不一样的，但实际上很多时候两个术语是可以互换的。

  • 锁：是一个具有获取和释放语句以维持特定属性的东西，比如：互斥锁，读写锁
  • 监视器：是一个机制，或者说是一个概念，用来保证在任何时候只有一个线程能够执行给定区块的代码。“监视器”可以通过“锁”来实现，但“监视器”的含义大于“锁”的含义。

Java中的监视器支持两种线程同步方式：互斥和协作。

  • 互斥：在Java虚拟机中，线程互斥是通过对象锁完成的。互斥可以使线程独立使用共享的数据，而不其它线程干扰。
  • 协作：在Java虚拟机中，线程协作是通过Object类中的wait和notify方法完成的。协作可以使多个线程为了同一个目标一起工作。

Java中的监视器

可以把监视器理解为一个建筑，这个建筑里有一个特殊的房间，这个房间同一时刻只能被一个线程所占有。这个房间里有一些数据。一个线程从进入该房间到离开该房间，可以全程独占享有该房间的所有数据。进入该建筑叫做进入监视器（entering the monitor），进入该房间叫做获得监视器（acquiring the monitor），独自占有该房间叫做拥有监视器（owning the monitor），离开该房间叫做释放监视器（releasing the monitor），离开该建筑叫做退出监视器（exiting the monitor）。

为了和数据打交道，一个监视器必须关联一些代码。一个监视器区域（一段代码）是一个不可分割的操作，即一个线程在执行监视器区域代码时，不允许其它线程同时执行该监视器区域代码。监视器需要保证同一时间只有一个线程在执行它的监视区域代码。一个线程想要进入该监视器，必须执行到该监视器区域代码的开始处，而该线程如果想要继续前进以执行监视器区域代码，则需要先获得该监视器。

当线程执行到该监视器区域时，会被放进监视器的入口区域（entry set），入口区域（entry set）相当于建筑里面的走廊，这时候相当于进入监视器。

  1. 如果入口区域中没有其它线程，并且没有其它线程占有监视器，那么该线程就可以获取监视器，并继续执行监视器区域代码。当该线程执行完监视器区域代码，则释放和退出监视器。
  2. 如果入口区域有其它线程，并且此时有一个线程正占有监视器，则该线程必须在入口区域等待，当占有监视器的线程退出监视器，则新来的线程必须和其它在入口区域中的线程进行竞争，竞争成功的线程才能获取监视器。

互斥方式可以避免线程使用共享数据时被其它线程干扰，而协作方式则帮助多个线程共同完成同一个目标。

当一个线程需要某些状态下的数据时，协作方式就变得很重要。举个例子，如果有两个线程，一个读线程，一个写线程，读线程读出写线程写入缓存的数据。如果缓存区没有数据，则读线程必须等待，等到写线程写入数据之后，读线程才能正常工作。

这种监视器通常被叫做等待和通知监视器（a “Wait and Notify” monitor），在这种类型的监视器中，占有当前监视器的线程可以通过执行等待命令（a wait command）来挂起自己，进入等待区（wait set），进入等待区的线程一直等到该监视器的其它线程执行通知命令（a notify command）才重新被激活。当该监视器里的一个线程执行了通知命令，执行通知命令的线程会继续占有该监视器，直到该线程主动释放该监视器（完成执行监视器区域代码或执行了等待命令进入了等待区）。当发出通知的线程释放了该监视器，在等待区等待的线程才会被复活，重新将会获得监视器。

图中有3个矩形，中间的大矩形仅包含一个线程（占有监视器的线程），左边的小矩形是入口区域（entry set），右边的小矩形是等待区（wait set）。5个标了序号的小门，代表了线程与监视器交互的过程。

整个过程是这样的：

  1. 当一个线程执行到监视器区域代码开始处时，穿过1号门进入监视器入口区域。
  2. 如果入口区域中没有其它线程，并且没有其它线程占有监视器，那么该线程就立即穿过2号门以占有监视器，这时候该线程可以继续执行监视器区域代码。如果此时监视器被其它线程占有，该线程则必须在入口区域等待，也因此被阻塞无法继续执行监视器区域代码。
  3. 如果占有监视器的线程发出等待命令，则穿过3号门释放监视器并进入等待区。
  4. 如果等待区的线程竞争成功，成功占有监视器，则通过4号门占有监视器，并继续执行监视器区域代码。
  5. 当占有监视器的线程执行完监视器区域代码，则穿过5号门释放并退出监视器。

注意

  1. 如果占有监视器的线程未发出过通知命令，则释放并退出监视器后，只有入口区域的线程会竞争获取监视器
  2. 如果占有监视器的线程发出过通知命令，则释放并退出监视器后，入口区域的线程会同一个或多个等待区的线程共同竞争获取监视器
  3. Java虚拟机提供了2个通知命令：notify()和notifyAll()。其中notify()方法唤醒任意一个等待区中的线程，notifyAll()方法唤醒等待区中的全部线程。
  4. Java虚拟机从下一个等待区或入口区选择下一个占有监视器的线程的算法由虚拟机的具体实现决定。
